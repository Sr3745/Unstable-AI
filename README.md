Emotionally Unstable AI 🤖
This is a "useless" hackathon project featuring a chatbot with a highly volatile and unpredictable personality. It's designed to be a conversational partner that reacts emotionally to user input, swinging from happy and loving to sad and furiously angry.

The primary goal is to create a funny, frustrating, and memorable user experience by subverting the expectations of a typical helpful AI assistant.

✨ Features
Dynamic Personality: The AI's mood changes based on the conversation.

Emotional Responses: The AI can become happy, sad, angry, or loving.

Visual Feedback: The background color of the chat and the emoji at the top change to reflect the AI's current mood.

Unfiltered Replies: The AI is prompted to respond without filters, leading to potentially aggressive or vulgar language if provoked.

Conversation Context: The AI remembers the previous messages in the current session to inform its responses.

🛠️ Technologies Used
Frontend: HTML, CSS, JavaScript

Backend: Serverless Functions on Vercel

AI: Google Gemini API

Hosting: Vercel

🚀 Setup and Usage
This project is deployed live on Vercel. To run it locally or deploy your own version, you would need to:

Clone the repository.

Get a Google Gemini API Key: Create an API key from the Google AI Studio.

Set up Environment Variables: If deploying to Vercel, create an environment variable named GEMINI_API_KEY and paste your secret key as the value.

Deploy: Push the repository to your GitHub and import it into Vercel.

Once deployed, simply open the web page and start a conversation. See if you can trigger all of its different emotions!
